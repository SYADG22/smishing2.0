SMS Message Classification API Documentation

Introduction

The SMS Message Classification API is designed to classify SMS messages as either "Smishing" (fraudulent) or "Legitimate" based on their content. It uses a pre-trained machine learning model to make predictions. This documentation provides details on how to use the API, including endpoints, request and response formats, and examples.

Base URL

The base URL for the API is:

http://your-api-domain.com

Authentication

The API does not require authentication for access.

Endpoint

GET /detect Description This endpoint allows you to classify an SMS message as "Smishing" or "Legitimate." Request Format Method: GET URL: /detect Headers: None Parameters: sms_message (JSON Object): sms_message (string, required): The text content of the SMS message to be classified.

Example Request

GET http://your-api-domain.com/detect?sms_message={"sms_message": "This is a sample SMS message for classification."}

Response format

Status Codes: 200 OK: Successful classification. 400 Bad Request: Missing or invalid input data. Body:

Example Response (Successful)

{ "result": "Legitimate" }

Example Response (Invalid Input)

{ "error": "Missing 'sms_message' field in the request" }

Usage

Make a GET request to the /detect endpoint with the sms_message parameter to classify an SMS message. The API will return the classification result as "Smishing" or "Legitimate."

Error Handling

If the sms_message parameter is missing in the request, the API will return a 400 Bad Request response with an error message

Deployment

To deploy this API, follow these steps: Set up a server or hosting environment for your API. Install the necessary dependencies, including Flask, scikit-learn, and any other required libraries. Deploy the API code to your server. Configure your web server (e.g., Nginx or Apache) to proxy requests to your Flask application. Ensure that the machine learning model and any required data files are accessible to the API. Start your API application.

Testing

To test the API, you can use tools like curl or Postman to make HTTP requests to the /detect endpoint with sample SMS messages.

Conclusion

The SMS Message Classification API allows you to classify SMS messages as "Smishing" or "Legitimate." It is a simple and effective tool for identifying potentially fraudulent messages in your application or system.
